# Agent Infra：AI Agent的六层基建框架

## Agent定义
能够带领人类并且自主行动完成目标的AI系统，具备感知能力、记忆、自主决策和行动能力。

## 六层Agent框架模型

### 1. 模型层（大脑）
- **作用**: 思考、推理、决策
- **代表**: GPT-5、Claude 4.1、千问3、DeepSeek V3
- **特点**: 只负责想，不负责做
- **输出**: 意图指令（如JSON格式的action）

### 2. 调度层（神经中枢）
- **作用**: 连接思考和行动的桥梁
- **功能**:
  - 解析指令
  - 分解任务
  - 按顺序发给不同组件
- **框架**: LangGraph、LlamaIndex、CrewAI
- **国内**: 阿里千问Agent

### 3. 记忆层
| 类型 | 说明 |
|------|------|
| 内置记忆 | 训练阶段学到的世界知识，固化在参数中 |
| 短期会话记忆 | 上下文窗口，调度层打包历史记录发给模型 |
| 中期工作记忆 | 任务暂存区，存储中间结论 |
| 长期个性化记忆 | 用户偏好、个人信息 |
| 外部记忆库 | RAG检索增强生成，可翻阅的参考书 |

- **工具**: MemoGPT、Memo0

### 4. 工具层（手脚）
- **演进路径**: 单一API → MCP协议 → 浏览器/电脑操作 → 云端专用工作环境
- **MCP协议**: 标准化工具调用，减少重复造轮子
- **Agent Bay**: 云电脑、云手机、浏览器、代码空间
  - 预装工具环境，开箱即用
  - 上下文持久化保存
  - 自净化系统（强化学习反馈）
  - 高并发支持

**代表产品**:
- AWS Agent Core
- 阿里云无影Agent Bay
- Kimi OK Computer
- 豆包Agent
- 智谱Auto GLM

### 5. 运维和治理层
**运维**:
- 高负载稳定运行
- 行为可观测
- 防止无意义算力消耗

**治理/安全**:
- 提示注入攻击
- 越权攻击
- 数据投毒
- 云服务比本地更安全（顶级云厂商防护）

### 6. 社会层（协作）
- **Multi-Agent架构**: 多Agent协作
- **A2A协议**: Agent间沟通协议
- **协作框架**: 类似人类的家庭、公司、组织架构

## 总结
Agent强大与否取决于六层基建：
1. 聪明的大脑（模型层）
2. 灵活的神经中枢（调度层）
3. 靠谱的记忆（记忆层）
4. 能干的手脚（工具层）
5. 健康的免疫系统（运维治理层）
6. 协作能力（社会层）
