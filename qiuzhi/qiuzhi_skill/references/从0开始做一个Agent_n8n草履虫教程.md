# 从0开始"做"一个Agent - N8N草履虫教程

## Agent定义
能够根据给定信息自己进行推理、规划和采取行动的AI智能系统。核心在于可以自己规划决定怎么做，然后自己用外部工具来做，并适应执行过程中的变化。

**简单说**：像一个能够思考、记忆和完成任务的数字员工。

## Agent vs 自动化工作流

| 自动化工作流 | Agent |
|------------|-------|
| 预先定义好的固定步骤 | 动态的、灵活的 |
| 遵循静态不变的规则 | 有自主推理能力 |
| 从A运行到B再到C的固定流程 | 动态决定怎么做、自主选择和操作工具 |
| 即使用了大模型也不是agent | 会根据情况调用工具 |

## Agent三大关键组件

### 1. 大脑（大语言模型）
- ChatGPT、Claude、Gemini、DeepSeek等
- 负责推理规划和语言生成

### 2. 记忆
- 记住之前的聊天和执行过程中的操作
- 利用背景和上下文做出更好决策
- 可能是过去N轮对话，或从文档/数据库提取信息

### 3. 工具
- **获取信息**：检索数据、搜索网页、从文档提取信息
- **采取行动**：发邮件、删数据库、创建日历行程、运行代码
- **编排**：调用其他代理、触发别的工作流

## N8N平台介绍
- 不需要写代码的可视化节点式搭建平台
- 德国创业公司，2019年开始，开源
- 在线版14天免费试用，本地安装完全免费
- 被称为"AI自动化的瑞士军刀"

## 实战案例：新闻早报Agent

### 功能
- 每天查看飞书多维表格里的AI新闻
- 查看日历日程
- 每天早上发新闻到飞书群
- 基于新闻和日程推荐内容

### 搭建步骤

#### 1. 安装N8N（Docker方式）
- 下载Docker桌面端
- 设置存储位置
- 运行命令拉取N8N

#### 2. 创建工作流
- 添加定时触发器（设置上午8点）
- 添加AI Agent节点

#### 3. 配置大脑
- 选择大模型（如DeepSeek）
- 添加API密钥

#### 4. 配置记忆
- 选择Simple Memory
- 设置记住N轮对话

#### 5. 配置工具
- 安装社区节点（飞书）
- 配置凭证（App ID、App Secret）
- 设置多维表格查询（过滤今天/昨天的新闻）
- 设置日历查询（往后七天日程）

#### 6. 配置输出
- HTTP请求往群里发消息
- POST请求 + Webhook URL
- 设置Content-Type: application/json

#### 7. 编写提示词
- 角色：是什么助手/专家
- 任务：整体任务概述
- 工具：可调用什么工具/数据
- 约束：遵循什么规则
- 输出：结果长什么样

## API和HTTP请求基础

### API（应用程序接口）
- 不同软件之间交流通信的方式
- 像ATM机：给格式正确的输入，得到响应

### HTTP请求
- **GET**：获取信息（天气、热搜等）
- **POST**：发送消息并接收结果

### JSON格式
- 请求中带的数据格式
- 大模型非常清楚如何处理

## 护栏（限制）
- 防止幻觉、陷入循环、不良行为
- 个人小项目问题不大
- 企业服务必须设置
- 案例：有人通过提示注入1块钱买走一辆车

## 核心原则
- 能用提示词解决就不用工作流
- 能用工作流就不用Agent
- 单个Agent能解决就不必用多个Agent
- 保持简单

## 提示词结构
1. 角色
2. 任务
3. 工具
4. 约束
5. 输出格式
