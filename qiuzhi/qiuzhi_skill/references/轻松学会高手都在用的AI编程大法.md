# 轻松学会高手都在用的AI编程大法

## 核心观点

**Spec Coding**是AI编程的新范式，核心理念是"先出规范的计划再干活"。亚马逊云科技发布的**Kilo IDE**是第一个有Spec模式的IDE，集成了敏捷开发、TDD、卡帕西AI编程原则等前沿范式。

## AI编程范式的演进

### 第一阶段：Type Coding
- 代码补全式辅助编程
- 主要靠Tab键无限续写

### 第二阶段：Vibe Coding（氛围编程）
- 随着Claude和Cursor的结合兴起
- 不用看懂代码，边聊边改边写
- 问题：只能做简单项目，项目复杂后bug改不动

### 第三阶段：Spec Coding
- 随着模型和Agent能力提升出现
- 核心理念：先出规范计划再干活
- 要求AI做更落地、更实用的开发

## Kilo IDE的Spec模式

### 三份核心文档

#### 1. 产品需求文档（PRD）
- 充当产品经理角色
- 拆解产品核心概念
- 定义产品哲学
- 专业术语表（定义代码中的对象和类）
- 用户故事定义功能
- 验收标准

#### 2. 技术设计文档
- 充当架构师角色
- 技术选型
- 接口设计
- 数据模型设计
- 错误处理策略

#### 3. 任务清单
- 充当项目经理角色
- 任务拆分顺序：基础架构 → 数据模型 → 状态管理 → UI组件
- 迭代交付、能分快跑通的开发线

## Spec模式的优势

- **开发质量高**：大量时间做测试和修复，交付结果基本能跑通
- **双向同步**：Spec文件与代码保持同步一致
- **能力集成**：包含大厂高P级别的产品设计、架构设计、项目管理能力
- **学习价值**：即使不用Kilo做项目，学习规范构建思路也很有价值

## 核心洞察

我们与AI的协作将变成：**如何用科学的工具、规范和流程去管理好这个超级同事**。
